# Project Structure

This guide explains the organization of the Quarto AI Slides repository.

```

├── _quarto.yml              # Root Quarto configuration

├── AI_AGENT_GUIDE.md        # Entry point for AI editors
├── quarto_install.sh        # Installation script
├── guides/                  # Documentation
│   ├── 00_quarto_basics.md
│   ├── 01_presentations_revealjs.md
│   ├── 02_presentations_pptx.md
│   ├── 03_project_structure.md
│   ├── 04_extensions_and_filters.md
│   ├── 05_ci_cd_github_actions.md

│   ├── executive-summary.qmd
│   ├── includes/            # Include files (partials)
│   └── theme/               # Theme assets
│       ├── slides.scss      # Custom SCSS
│       └── brand.potx       # PowerPoint template

│   ├── deck-executive-summary/
```

## Key Files

### `_quarto.yml`

Root configuration file that defines:
- Project type (website)
- Output directory (`docs/` for GitHub Pages)
- Default format options
- Website navigation

### Templates

Located in `templates/`:
- **revealjs-minimal.qmd**: Starter for HTML presentations
- **pptx-minimal.qmd**: Starter for PowerPoint
- **beamer-minimal.qmd**: Starter for PDF
- **executive-summary.qmd**: Pattern for executive summaries

### Examples

Located in `examples/deck-*/`:
- Real-world examples demonstrating best practices
- Copy these as starting points for new decks
- Each deck is in its own directory with `index.qmd`

### Guides

Located in `guides/`:
- Numbered guides (00-90) for logical ordering
- Reference documentation for AI agents and users
- Rendered as part of the website

## Creating New Decks

### Recommended Structure

```
examples/deck-your-name/
└── index.qmd
```

Or for more complex decks:

```
examples/deck-your-name/
├── index.qmd
├── images/
│   └── logo.png
└── data/
    └── data.csv
```

### Naming Convention

- Use kebab-case: `deck-weekly-sync`, `deck-product-launch`
- Be descriptive but concise
- Avoid special characters

## Output Directory

### `docs/` (Mode A - Default)

- Rendered HTML files go here
- GitHub Pages serves from this directory
- Keep this directory in version control
- Automatically generated by `quarto render`

### `gh-pages` Branch (Mode B - Optional)

- Alternative publishing method
- Requires workflow modification
- Keeps rendered files out of main branch

## Configuration Hierarchy

1. **`_quarto.yml`**: Project-wide defaults
2. **Individual `.qmd` files**: File-specific overrides
3. **Command-line options**: Runtime overrides

## Best Practices

1. **Keep templates generic**: Don't hardcode project-specific content
2. **Use examples as reference**: Copy structure, not content
3. **Organize by purpose**: Group related decks
4. **Document customizations**: Note any deviations from templates
5. **Version control**: Commit both source (`.qmd`) and rendered output (`docs/`)
